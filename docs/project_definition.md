# 项目定义

## V1 要做什么
- 建立以 LaTeX 文件索引为核心的题库主库。
- 建立标准化 bundle 格式，统一录入试卷主文件、题目文件、答案文件、图片和 xlsx 统计工作簿。
- 提供稳定的只读查询 API。
- 同时保存原始 xlsx 统计表和按题聚合统计。

## V1 不做什么
- 不保存考生个人原始成绩。
- 不做复杂后台管理页面。
- 不把生产 LaTeX、生产 xlsx 或服务器密钥提交进源码仓。
- 不在 V1 里实现 IRT 等需要逐题匿名作答数据的复杂模型。

## 源码仓职责
- 保存 schema、脚本、API、样例 bundle、测试和文档。
- 提供本地开发和验证所需的样例 LaTeX 与样例 xlsx。
- 通过环境变量或命令行参数连接服务器上的生产数据库。

## 服务器职责
- 保存生产 SQLite、生产 LaTeX、生产 xlsx、生产 assets 和运行日志。
- 运行 FastAPI 进程并对外提供只读查询接口。
- 承担备份、回滚和服务重启。

## 核心实体
- `paper`: 一套试卷，对应一个主 LaTeX 文件。
- `question`: 一道题目，对应一个题目 LaTeX 文件与可选答案 LaTeX 文件。
- `paper_index`: 题目在卷内的顺序索引。
- `score_workbook`: 一份原始 xlsx 统计工作簿，直接写入数据库。
- `question_stats`: 从原始工作簿或中间 CSV 提炼出的按题统计量。
